FAQ

Answers to common questions about Nova.

General

What is Nova?

Nova is a programming language designed for the AI age. It combines familiar Rust-like syntax with built-in formal verification, letting you write code that proves itself correct.

Nova compiles to WebAssembly (WASM), making it run anywhere: browsers, servers, edge functions, and embedded systems.

Is Nova ready for production use?

Not yet. Nova is in early development. The lexer and parser are complete (100% of tests passing), but type checking and code generation are still in progress.

Check our Status Page for an honest assessment of what works today.

Why create another programming language?

AI is writing more code every day, but it makes mistakes. Verification helps catch these mistakes before they cause problems.

Most verified languages are academic and hard to learn. Nova aims to be:

  • Familiar to Rust/TypeScript developers
  • Practical for real-world use
  • AI-friendly (clear syntax, good error messages)
What's the license?

Nova is open source under the MIT License. You can use it for any purpose, including commercial projects.

Technical

What does Nova's syntax look like?

Nova uses Rust-like syntax that should feel familiar:

fn add(a: i32, b: i32) -> i32 { a + b }

See the full Syntax Reference for all language constructs.

What does "verification" mean?

Verification means mathematically proving your code is correct. Instead of just testing, you prove properties like:

  • A function always returns a positive number
  • An array index is always within bounds
  • A division never divides by zero

This is done through preconditions (requires), postconditions (ensures), and invariants.

Why target WebAssembly?

WASM provides:

  • Portability - Run the same code everywhere
  • Security - Sandboxed execution by default
  • Performance - Near-native speed
  • Interoperability - Call from JavaScript, Rust, Python, etc.
How does Nova compare to Rust?

Nova is inspired by Rust but makes different tradeoffs:

  • Rust: Memory safety through ownership/borrowing
  • Nova: Correctness through formal verification

Nova doesn't have Rust's borrow checker. Instead, it uses verification to prove properties at compile time.

How does Nova compare to Dafny or F*?

Dafny and F* are excellent verified languages, but they're academic in nature. Nova aims to be:

  • More familiar syntax (Rust-like vs. custom)
  • Better tooling and error messages
  • Easier to learn for mainstream developers

Learning

How do I get started?

Start with our Learn section. It teaches how programming languages work by walking through Nova's implementation.

You don't need prior compiler experience - we start from the basics.

Do I need to know Rust?

No, but it helps. Nova's syntax is similar to Rust, so Rust knowledge transfers directly.

If you know TypeScript, C++, or any C-family language, you'll pick up Nova quickly.

Is there a playground to try Nova?

Not yet. An interactive playground is planned but requires working code generation, which is still in development.

For now, you can clone the repository and run the lexer/parser locally.
